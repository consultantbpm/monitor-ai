The user invoked `/tcm-context` to consolidate everything relevant about a topic into one markdown file. Use case: start a fresh Claude conversation (another worktree, another thread, Claude Desktop) with focused context instead of starting from zero.

## Parse the argument

- First positional: topic or free-text query. Required. If missing, ask: "Despre ce topic să fac bundle-ul?"
- `--max-tokens=N`: optional ceiling. Default 3000.

## What to collect

In this order:

1. **Notes matching the topic** (both project + global):
   - Files in `c:\AN\ai-code-exposure-monitor\.teachcm/notes/*.md` and `~/.teachcm/notes/*.md`.
   - Match by: title, tags, content keywords. Use simple substring + tag match (no fancy semantic search at this level).
   - Sort by relevance: tag match > title match > content match.

2. **CLAUDE.md sections** relevant to topic:
   - Read `<workspace>/CLAUDE.md`.
   - Split into sections by H2 (`##`).
   - Match section title or first paragraph against topic.

3. **Decisions** (ADRs) relevant to topic:
   - Files in `c:\AN\ai-code-exposure-monitor\.teachcm/decisions/*.md`.
   - Match title, tags, content.

4. **Recent mockups + breakdowns** if relevant:
   - `<workspace>/.tcm-mockups/<slug>*.html` matching topic slug.
   - `c:\AN\ai-code-exposure-monitor\.teachcm/breakdowns/<slug>*.md`.

## Token budget

Estimate tokens as `wordCount * 1.3` if no tokenizer available.

Stop adding items once estimated tokens exceeds `max-tokens`. Prefer higher-relevance items.

If skipped items exist due to budget, list them at the end as "Also available (skipped due to budget): <file paths>".

## Write the bundle

Path: `<workspace>/.teachcm/exports/context-<topic-slug>-<YYYYMMDD-HHMMSS>.md`

Create the `exports/` directory if missing.

